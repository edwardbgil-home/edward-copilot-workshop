# Task Manager CLI ‚Äì Technical Design

## 1. Data Models

### Task

The primary entity. Stored in-memory as a plain JavaScript object.

| Property      | Type     | Required | Validation rules                                      |
|---------------|----------|----------|-------------------------------------------------------|
| `id`          | `string` | Yes      | UUID v4 generated by `crypto.randomUUID()`. Read-only after creation. |
| `title`       | `string` | Yes      | Non-empty string after trimming whitespace.           |
| `description` | `string` | No       | Defaults to `''` when omitted. No length constraint.  |
| `status`      | `string` | Yes      | One of `'todo'`, `'in-progress'`, `'done'`. Defaults to `'todo'` on create. |
| `priority`    | `string` | Yes      | One of `'low'`, `'medium'`, `'high'`. Defaults to `'medium'` on create. |
| `createdAt`   | `string` | Yes      | ISO 8601 timestamp. Set once on create. Read-only after creation. |
| `updatedAt`   | `string` | Yes      | ISO 8601 timestamp. Set on create and refreshed on every update. |

### Property Validation Rules

The following rules are enforced by `src/validation/task-validation.js` before any mutation reaches the store.

**`id`**
- Generated internally by `crypto.randomUUID()`; never accepted from user input.
- Read-only: rejected if supplied in `TaskCreateInput` or `TaskUpdateInput`.

**`title`**
- Must be present in `TaskCreateInput`; absence or `undefined` throws an error.
- Must be of type `string`.
- Must be non-empty after calling `.trim()`; a whitespace-only string is rejected.
- Stored after trimming leading and trailing whitespace.
- On update, if provided, the same non-empty-after-trim rule applies.

**`description`**
- Optional in both `TaskCreateInput` and `TaskUpdateInput`.
- Must be of type `string` when provided; `null` or non-string values are rejected.
- Defaults to `''` on create when omitted or `undefined`.
- No maximum length constraint.

**`status`**
- Must be one of the values exported in `STATUSES`: `'todo'`, `'in-progress'`, `'done'`.
- Defaults to `'todo'` on create when omitted.
- On update, if provided, must be one of the three allowed values; any other string throws an error.
- Case-sensitive: `'TODO'` or `'In-Progress'` are rejected.

**`priority`**
- Must be one of the values exported in `PRIORITIES`: `'low'`, `'medium'`, `'high'`.
- Defaults to `'medium'` on create when omitted.
- On update, if provided, must be one of the three allowed values; any other string throws an error.
- Case-sensitive: `'Low'` or `'HIGH'` are rejected.

**`createdAt`**
- Set once by `createTask()` using `now()` (`new Date().toISOString()`).
- Read-only: rejected if supplied in `TaskCreateInput` or `TaskUpdateInput`.
- Format is always ISO 8601 (e.g., `'2026-03-31T12:00:00.000Z'`).

**`updatedAt`**
- Set to the same value as `createdAt` when the task is first created.
- Refreshed to `now()` on every successful `updateTask()` call.
- Read-only: rejected if supplied in `TaskUpdateInput`.
- Format is always ISO 8601.

### Supporting Types (JS value conventions)

| Name              | Shape                                                      | Used by                     |
|-------------------|------------------------------------------------------------|-----------------------------|
| `TaskStatus`      | `'todo' \| 'in-progress' \| 'done'`                       | `task.js`, `task-validation.js`, `enums.js` |
| `TaskPriority`    | `'low' \| 'medium' \| 'high'`                             | `task.js`, `task-validation.js`, `enums.js` |
| `TaskCreateInput` | `{ title: string; description?: string; status?: TaskStatus; priority?: TaskPriority }` | `task-service.js`, `task-validation.js` |
| `TaskUpdateInput` | `{ title?: string; description?: string; status?: TaskStatus; priority?: TaskPriority }` | `task-service.js`, `task-validation.js` |
| `ListOptions`     | `{ filterStatus?: TaskStatus; filterPriority?: TaskPriority; sortBy?: 'priority'\|'createdAt'; sortOrder?: 'asc'\|'desc' }` | `task-service.js`, `task-validation.js` |

## 2. File Structure

```text
src/
  cli/
    index.js              # Entry point; parses process.argv and routes to commands
    help.js               # Builds and prints usage / help text to stdout
  constants/
    enums.js              # Exports STATUSES, PRIORITIES, and SORT_PRIORITY_ORDER arrays/maps
  models/
    task.js               # Exports createTask() factory; sets id, timestamps, and defaults
  services/
    task-service.js       # Exports CRUD functions and list() with filtering and sorting; owns the in-memory store
  validation/
    task-validation.js    # Exports validateCreateInput(), validateUpdateInput(), validateListOptions()
  utils/
    format.js             # Exports formatTask() and formatTaskList() for terminal output
    time.js               # Exports now() returning the current ISO 8601 timestamp
test/
  task.test.js            # Unit tests for task.js (model factory)
  task-service.test.js    # Unit tests for task-service.js (CRUD, filter, sort)
  task-validation.test.js # Unit tests for task-validation.js (valid and invalid inputs)
docs/
  project-plan.md         # Project plan (produced by Planner Agent)
  schema.md               # This file (produced by Architect Agent)
```

## 3. Module Responsibilities

### `src/constants/enums.js`

**Exports:**
- `STATUSES` ‚Äì `string[]` of valid status values
- `PRIORITIES` ‚Äì `string[]` of valid priority values
- `SORT_PRIORITY_ORDER` ‚Äì `Object` mapping priority to numeric weight for sorting

**Depends on:** nothing

---

### `src/utils/time.js`

**Exports:**
- `now()` ‚Äì returns `new Date().toISOString()`

**Depends on:** nothing

---

### `src/utils/format.js`

**Exports:**
- `formatTask(task)` ‚Äì returns a single-line string representation of a task
- `formatTaskList(tasks)` ‚Äì returns a multi-line string or a no-tasks message

**Depends on:** nothing

---

### `src/models/task.js`

**Exports:**
- `createTask(input)` ‚Äì accepts `TaskCreateInput`, merges defaults, generates `id` and timestamps, returns a `Task`

**Depends on:** `src/utils/time.js`, `src/constants/enums.js`

---

### `src/validation/task-validation.js`

**Exports:**
- `validateCreateInput(input)` ‚Äì throws `Error` if `title` is missing/empty or enum fields are invalid
- `validateUpdateInput(input)` ‚Äì throws `Error` if no fields are provided or enum fields are invalid
- `validateListOptions(options)` ‚Äì throws `Error` if filter values or sort options are invalid

**Depends on:** `src/constants/enums.js`

---

### `src/services/task-service.js`

**Exports:**
- `addTask(input)` ‚Äì validates input, creates and stores a task, returns the new `Task`
- `getTask(id)` ‚Äì returns the matching `Task` or throws a not-found `Error`
- `listTasks(options?)` ‚Äì returns a filtered and sorted `Task[]`
- `updateTask(id, input)` ‚Äì validates input, merges changes, refreshes `updatedAt`, returns the updated `Task`
- `deleteTask(id)` ‚Äì removes the task or throws a not-found `Error`

**Internal state:** a module-level `tasks` array (`Task[]`)

**Depends on:** `src/models/task.js`, `src/validation/task-validation.js`, `src/constants/enums.js`, `src/utils/time.js`

---

### `src/cli/help.js`

**Exports:**
- `printHelp()` ‚Äì writes usage instructions to `stdout`

**Depends on:** nothing

---

### `src/cli/index.js`

**Exports:** none (entry point, side-effectful)

Parses `process.argv`, dispatches to the appropriate `task-service` function, formats output with `format.js`, and calls `printHelp()` for unknown commands.

**Depends on:** `src/services/task-service.js`, `src/utils/format.js`, `src/cli/help.js`

## 4. Error Handling Strategy

All errors are `Error` instances with descriptive messages. Plain strings are never thrown.

| Error type          | Message pattern                                        | Thrown in                      |
|---------------------|--------------------------------------------------------|--------------------------------|
| Validation ‚Äì missing field   | `'title is required and must be a non-empty string'`  | `validateCreateInput()`        |
| Validation ‚Äì invalid enum    | `'status must be one of: todo, in-progress, done'`    | `validateCreateInput()`, `validateUpdateInput()`, `validateListOptions()` |
| Validation ‚Äì no fields       | `'update input must include at least one field'`      | `validateUpdateInput()`        |
| Validation ‚Äì invalid sort    | `'sortBy must be one of: priority, createdAt'`        | `validateListOptions()`        |
| Not found                    | `'Task not found: <id>'`                              | `getTask()`, `updateTask()`, `deleteTask()` |

### Conventions

- `src/validation/task-validation.js` is the single location where input shape errors are thrown.
- `src/services/task-service.js` catches no errors internally; callers (i.e. `src/cli/index.js`) are responsible for wrapping service calls in `try/catch`.
- `src/cli/index.js` wraps every command dispatch in a `try/catch`, logs the `error.message` with `console.error`, and exits with code `1` on failure.
- Timestamps and ID generation do not throw; no error handling is needed for those utilities.
